# Report Format Specification

## Overview

TEP generates a `RESULT.md` file in the output directory after each audit.
This document describes the structure and conventions used.

## File Location

- Default: `out/RESULT.md` (relative to TEP repo root)
- Override: `--out <dir>` flag
- Written atomically (temp file → mv) to avoid partial writes

## Structure

# TEP Audit Report — <project_name>
```
> Generated on <timestamp> by `tep audit`
>
> Project: `<project_dir>`
>
```

## Section Details

### 1. Inventory
Table listing all detected Claude Code config files with their status (found/missing).

### 2. Audit CLAUDE.md
- Size metrics (words, tokens, sections)
- Volatile content detection
- Cache-friendly order assessment
- Hardcoded paths

### 3. Budget tokens
Table with per-file token estimates (lines, chars, ~tokens).
Total config token cost.

### 4. Doublons
Cross-reference analysis between CLAUDE.md and skills.
Risk count and details.

### 5. Ignore Files
.gitignore and .claudeignore issue counts and details.

### 6. Hooks & Settings
Hook inventory, MCP server count, safety analysis.

### 7. Session Stats
If available: token breakdown, cache hit rate, cost estimate, top files/actions.
If not: reason why (no session, no jq).

### 8. Score & Recommandations
8-criteria scoring table + personalized recommendations.

### 9. Historique
Entry count, trend (improved/degraded/stable).

## Markers

The report uses emoji markers for status:
- ✅ (`TEP_MD_OK`) — Pass
- ⚠️ (`TEP_MD_WARN`) — Warning
- ❌ (`TEP_MD_FAIL`) — Fail
- ℹ️ (`TEP_MD_INFO`) — Info

## History Format (evals/history.jsonl)

Each line is a JSON object:

```
{
    "timestamp": "2025-01-15 14:30",
    "project": "my-app",
    "score": 6,
    "max_score": 8,
    "claude_md_tokens": 1200,
    "total_config_tokens": 2500,
    "cache_hit_rate": 85,
    "cost_usd": "0.0342",
    "duplicates": 0,
    "hardcoded_paths": 0,
    "volatile_types": 1,
    "hooks_issues": 0,
    "skills": 3,
    "rules": 0,
    "mcp_servers": 2,
    "order_ok": true
}
```

The file grows by one line per audit. Use `jq` to query:

# Last 5 scores
```
tail -5 evals/history.jsonl | jq '.score'
```
# Score trend
```
jq -s 'map(.score)' evals/history.jsonl
```
# Average cache hit rate
```
jq -s 'map(.cache_hit_rate) | add / length' evals/history.jsonl
```
